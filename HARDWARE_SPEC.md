# ESP32 四足机器马项目 - 硬件清单与架构设计（1-DOF 四舵机版本）

> 本文档对应 **入门版 1-DOF 四足机器马**，每条腿仅 1 个舵机控制大腿前后摆动，小腿通过被动连杆跟随。适合零基础初学者快速验证步态逻辑。

---

## 1. 硬件清单

### 主控核心
- **ESP32-S3-DevKitC-1-N16R8 开发板**
  - 16MB Flash + 8MB PSRAM
  - WiFi + 蓝牙 5.0
  - 支持 DVP 摄像头、AI 加速、多外设

### 执行器（核心改动：仅 4 个舵机）
- **舵机 × 4（MG996R 或标准模拟舵机）**
  - 左前腿（LF）— GPIO 1
  - 右前腿（RF）— GPIO 2
  - 左后腿（LH）— GPIO 42
  - 右后腿（RH）— GPIO 41
  - **每条腿仅 1-DOF**：舵机控制大腿前后摆动
  - 小腿通过**机械连杆被动跟随**（无需舵机）

### 视觉模块
- **140° 广角 OV2640 摄像头模块（24Pin DVP 接口）**
  - 200 万像素
  - 1/4 英寸靶面
  - 广角大视野，适合四足避障

### 姿态传感器（9 轴带磁力计）
- **GY-9250 模块（MPU9250 + AK8963 磁力计）**
  - 9 轴：加速度计 + 陀螺仪 + 电子罗盘
  - 内置 DMP 硬件姿态解算
  - 可直接输出 Pitch / Roll / Yaw

### 显示设备
- **SPI 接口 IPS 彩屏（ST7789 或 ILI9341 驱动）**
  - 用于显示摄像头画面、姿态角、调试信息

### 供电配件
- **DC 圆孔电源母头转杜邦线（公头）**
  - 线材规格：22AWG / 24AWG 纯铜电源线
  - 计划供电：5V 3A 直流电源（4 舵机起步够用）
- **100μF 电解电容 × 1**（并联在舵机电源轨上稳压）

### 连接线材
- **杜邦线（公对公、公对母）**
  - 用于传感器、屏幕、舵机信号线与 ESP32 接线
- **面包板 × 3（MB-102）**
  - 面包板 1：ESP32 主控 + 传感器/显示屏
  - 面包板 2：舵机独立电源区
  - 面包板 3：扩展/备用/摄像头测试区

## 3. 四足机器人运动学架构（1-DOF）

### 3.1 腿部结构设计
每条腿 **1 个自由度（1-DOF）**：
- **大腿关节**：舵机直接控制大腿前后摆动
- **小腿**：通过**被动连杆**（机械耦合）跟随大腿运动
  - 大腿向前摆 → 连杆拉动小腿离地（摆动相）
  - 大腿向后摆 → 连杆放下小腿触地（支撑相）

### 3.2 运动方式
- ✅ 只能走直线（前进/后退）
- ❌ **不能转向**（缺少髋关节自由度）
- ❌ **不能调节身体高度**（缺少膝关节自由度）

### 3.3 步态规划
| 步态 | 描述 | 稳定性 |
|------|------|--------|
| **爬行步态（Crawl）** | 每次移动一条腿，三角支撑 | ⭐⭐⭐⭐⭐ |
| **小跑步态（Trot）** | 对角腿同步摆动 | ⭐⭐⭐⭐ |
| **奔跑步态（Gallop）** | 前后腿交替快速移动 | ⭐⭐⭐ |

> 入门阶段先实现**爬行步态**即可让机器马走起来。

## 4. 软件架构设计（Arduino IDE）

### 4.1 项目目录结构

相比 12 舵机版本**大幅简化**，无需逆运动学：

```
ESP32_Quadruped_Horse/
├── ESP32_Quadruped_Horse.ino    # 主程序入口
├── config.h                     # 硬件配置与引脚定义
├── motion/
│   ├── gait_generator.h/.cpp    # 步态生成器（简单查表/三角函数）
│   └── servo_controller.h/.cpp  # 舵机控制器（4路PWM）
├── sensors/
│   ├── mpu9250_driver.h/.cpp    # MPU9250 姿态传感器驱动
│   └── camera_driver.h/.cpp     # OV2640 摄像头驱动
├── display/
│   └── lcd_driver.h/.cpp        # 显示屏驱动
└── utils/
    └── lowpass_filter.h/.cpp    # 低通滤波器（可选）
```

### 4.2 核心模块依赖关系
```
main.ino
  ├── config.h (全局配置)
  ├── motion/
  │   ├── gait_generator (步态规划 — 直接输出4个角度)
  │   └── servo_controller (舵机输出 — servo.write())
  ├── sensors/
  │   ├── mpu9250_driver (姿态反馈)
  │   └── camera_driver (视觉输入)
  └── display/
      └── lcd_driver (状态显示)
```

### 4.3 主循环架构
```cpp
void loop() {
    // 1. 读取传感器数据（100Hz）
    readSensors();
    
    // 2. 更新姿态估计（100Hz）
    updateAttitude();
    
    // 3. 视觉处理（10-15fps）
    processVision();
    
    // 4. 步态生成 — 直接计算4个舵机角度（50Hz）
    generateGait();  // 无需逆运动学！
    
    // 5. 输出舵机控制信号（50Hz）
    updateServos();  // 4行 servo.write()
    
    // 6. 更新显示（10fps）
    updateDisplay();
}
```

## 5. 开发阶段规划（1-DOF 路线）

### Phase 0：让机器马走起来（当前目标）
- [ ] 搭建面包板，4 路舵机独立供电
- [ ] 上传舵机测试代码，确认 4 个舵机依次摆动正常
- [ ] 编写基础步态（爬行），让马能直线前进
- [ ] 调试步态参数（角度范围、速度、停顿时间）

### Phase 1：基础驱动开发
- [ ] MPU9250 传感器驱动与 DMP 姿态解算
- [ ] SPI 显示屏驱动（显示当前角度、姿态角）
- [ ] OV2640 摄像头基础采集
- [ ] 整合：显示屏实时显示步态状态

### Phase 2：增强功能
- [ ] 基于 MPU9250 的姿态稳定（检测到倾斜自动调整步态）
- [ ] 蓝牙/WiFi 遥控（手机 APP 控制走/停/速度）
- [ ] 视觉避障（摄像头检测到障碍停止或绕行）
- [ ] 多种步态切换（爬行/小跑）

### Phase 3：升级 2-DOF（可选）
- [ ] 每条腿增加 1 个膝关节舵机（共 8 个舵机）
- [ ] 实现转向功能（左右腿差速）
- [ ] 实现身体高度调节
- [ ] 引入简单逆运动学（2-DOF 平面解算）

### Phase 4：终极 3-DOF（远景）
- [ ] 每条腿 3 个舵机（共 12 个）
- [ ] 完整 3-DOF 逆运动学
- [ ] 复杂地形适应（爬坡、越障）

## 7. 注意事项与建议

### 电源管理
- 4 个 MG996R 舵机，**5V/3A 电源勉强够用**（单个堵转电流约 1.2A，4 个同时启动峰值较高）
- **建议**：实际运行时先一次只测试 1-2 个舵机，观察电源是否发热
- 舵机电源线使用 **22AWG 粗线**，信号线使用 24AWG
- **100μF 电解电容必须并联在舵机电源轨上**（长腿+接 5V，短腿-接 GND）

### 面包板布局
- **面包板 1**：ESP32 插在中间区域，上方接 MPU9250/显示屏
- **面包板 2**：专门给舵机供电，左侧红轨=5V，左侧蓝轨=GND
- **两块面包板的蓝轨（GND）必须用粗跳线连通**（共地！否则舵机不受控）

### 信号完整性
- 4 路舵机信号线直接从 GPIO 飞线到舵机，无需 PCA9685 驱动板
- I2C 总线（MPU9250）建议上拉电阻 4.7kΩ（接在 3.3V 和 SDA/SCL 之间）

### 机械结构建议
- 舵机安装在大腿根部，输出轴连接大腿连杆
- 小腿通过**四连杆机构**与大腿耦合，实现被动跟随
- 腿部连杆比例建议：大腿 : 小腿 ≈ 1 : 1.2
- 总重量控制在 1.5kg 以内，单腿负载不超过 400g

### 调试技巧
- 先用舵机测试代码逐个确认 4 个舵机摆动方向一致
- 测试单腿步态：让一个舵机反复摆动，观察机械连杆是否正常工作
- 串口输出 4 个舵机的实时角度，使用 Serial Plotter 观察波形
- 步态调试顺序：先静止站立（全部 90°）→ 单腿摆动 → 对角腿联动 → 四腿协调
