# TFT_eSPI + GT30L32S4W 汉字显示（硬件 SPI 共享总线版）

> 基于硬件 SPI 共享总线的最终实现，使用 TFT_eSPI 库替代 Adafruit_ST7789，性能更高。
>
> 核心原理：TFT_eSPI 和字库芯片共享 ESP32-S3 的 FSPI 总线（GPIO11/12/13），通过 CS 片选分时复用。

---

## 一、与 Adafruit 硬件 SPI 版的对比

| 项目 | Adafruit 硬件 SPI | TFT_eSPI 硬件 SPI |
|------|------------------|-------------------|
| 库体积 | 较大 | 更轻量 |
| 刷新速度 | 40MHz | 40MHz（支持 DMA）|
| 绘制效率 | `drawRGBBitmap` | `pushImage`（更快）|
| 初始化 | `tft.init(240, 240)` | `tft.init()` |
| 反色处理 | 手动 `invertDisplay(true)` | 自动处理（默认正确）|
| 中文显示 | 支持 | 支持（代码相同）|
| 引脚配置 | 代码中传入 | `User_Setup.h` 中配置 |

## 二、为什么 TFT_eSPI 也能共享 SPI 总线

### 2.1 共享总线的关键：`SPI.beginTransaction`

```
ESP32-S3 FSPI 总线
┌─────────────────────────────────────┐
│  GPIO11 (MOSI) ──────┬──→ 屏幕 SDA  │
│  GPIO12 (SCK)  ──────┤──→ 屏幕 SCL  │
│  GPIO13 (MISO) ──────┤──→ 字库 FSO  │
│  GPIO10 (CS1)  ──────┘──→ 屏幕 CS1  │
│  GPIO9  (CS2)  ─────────→ 字库 CS2  │
└─────────────────────────────────────┘
```

每次 `SPI.beginTransaction(SPISettings(...))` 会：
1. **锁定 SPI 总线**（防止中断插入）
2. **重新配置 SPI 寄存器**（速度、模式、位序）
3. **本次操作完成后**，`endTransaction()` 释放总线

### 2.2 时序串行性保证安全

```
时间线：
├─ TFT_eSPI 操作 ─┤├─ 字库读取 ─┤├─ TFT_eSPI 操作 ─┤
  CS1=0, CS2=1       CS1=1, CS2=0      CS1=0, CS2=1
  beginTransaction   beginTransaction   beginTransaction
  ...SPI.transfer... ...SPI.transfer... ...SPI.transfer...
  endTransaction     endTransaction     endTransaction
```

**任意时刻只有一个设备的 CS 为低**，总线参数由 `beginTransaction` 即时切换，不会冲突。

### 2.3 为什么不能用 `digitalWrite`

`SPI.begin()` 后，GPIO11/12/13 被路由给 SPI 外设：

```
┌──────────────┐
│  GPIO Matrix  │  GPIO11/12/13 控制权转移
│  (IO_MUX)    │
└──────┬───────┘
       │
┌──────▼────────┐
│   SPI 外设    │  ← 现在由这里控制引脚
│  (FSPI/HSPI)  │
└───────────────┘
```

`digitalWrite(SCK, HIGH)` → **请求被忽略**，因为引脚所有权在 SPI 外设。

所以字库读取必须走 `SPI.transfer()`，不能用 bit-bang。

## 三、配置 User_Setup.h

TFT_eSPI 的引脚配置不在代码中，而在库目录的 `User_Setup.h` 中。

### 3.1 文件位置

```
Documents/Arduino/libraries/TFT_eSPI/User_Setup.h
```

### 3.2 关键配置项

```cpp
// ========== 驱动选择 ==========
#define ST7789_DRIVER

// ========== 屏幕尺寸 ==========
#define TFT_WIDTH  240
#define TFT_HEIGHT 240

// ========== SPI 引脚（ESP32-S3 默认 FSPI）==========
#define TFT_MISO  13   // FSO（字库回读）
#define TFT_MOSI  11   // SDA（屏幕+字库共用）
#define TFT_SCLK  12   // SCL（屏幕+字库共用）
#define TFT_CS    10   // CS1（屏幕片选）
#define TFT_DC     6   // DC（数据/命令）
#define TFT_RST    7   // RES（复位）

// ========== 背光 ==========
#define TFT_BL     5   // BLK（可选，代码中手动控制）

// ========== 加载字体（可选）==========
// #define LOAD_GLCD    // 标准 Adafruit 字体
// #define LOAD_FONT2   // 小型字体
// #define LOAD_FONT4   // 中型字体
// #define LOAD_FONT6   // 大型字体
// #define LOAD_FONT7   // 7段字体
// #define LOAD_FONT8   // 平滑字体
// #define LOAD_GFXFF   // Adafruit GFX 字体

// ========== SPI 频率 ==========
#define SPI_FREQUENCY  40000000   // 40MHz 写屏
```

### 3.3 重要：关闭其他 Setup 文件

TFT_eSPI 库可能有多个 Setup 文件，确保只启用一个：

```cpp
// 在 User_Setup.h 顶部确保有：
// #include <User_Setup.h>   ← 这是默认的

// 不要启用：
// #include <User_Setups/SetupX_xxx.h>  ← 注释掉这些
```

## 四、代码解析

### 4.1 字库读取（共享总线安全）

```cpp
bool fontRead(uint32_t addr) {
  // 防御性拉高屏幕 CS（TFT_eSPI 通常已处理，但保险）
  digitalWrite(TFT_CS, HIGH);

  // 配置 SPI 为字库参数
  SPI.beginTransaction(SPISettings(8000000UL, MSBFIRST, SPI_MODE0));
  digitalWrite(FONT_CS, LOW);

  // 发送 READ 指令和 24 位地址
  SPI.transfer(0x03);
  SPI.transfer((addr >> 16) & 0xFF);
  SPI.transfer((addr >>  8) & 0xFF);
  SPI.transfer( addr        & 0xFF);

  // 读取 32 字节点阵
  for (int i = 0; i < 32; i++) {
    fontBuf[i] = SPI.transfer(0x00);
  }

  digitalWrite(FONT_CS, HIGH);
  SPI.endTransaction();  // 释放总线
  // ...
}
```

### 4.2 绘制汉字（pushImage）

```cpp
// 关键：缓冲区必须 4 字节对齐！
static uint16_t pixBuf[16 * 16] __attribute__((aligned(4)));

void drawHanzi(int16_t x, int16_t y, uint8_t msb, uint8_t lsb,
               uint16_t fg, uint16_t bg) {
  // 1. 读取字库
  uint32_t addr = gb2312Addr(msb, lsb);
  if (!addr || !fontRead(addr)) return;

  // 2. 解码到像素缓冲
  for (int row = 0; row < 16; row++) {
    // ... 解码逻辑 ...
  }

  // 3. 一次性推送（内部自动处理 SPI 事务）
  // pushImage 内部已调用 begin_tft_write/end_tft_write
  // 不需要外部包裹 startWrite/endWrite
  tft.pushImage(x, y, 16, 16, pixBuf);
}
```

**⚠️ 关键：缓冲区 4 字节对齐**

ESP32-S3 的 SPI 外设通过 `pushPixels` 发送数据时，将缓冲区强制转换为 `uint32_t*` 进行 32-bit DMA 传输。如果缓冲区未 4 字节对齐，会触发硬件未对齐访问异常，导致数据发送失败。

```cpp
// ❌ 错误：仅 2 字节对齐
static uint16_t pixBuf[16 * 16];

// ✅ 正确：强制 4 字节对齐
static uint16_t pixBuf[16 * 16] __attribute__((aligned(4)));
```

**`pushImage` 的优势**：
- 内部只需 1 次 `setAddrWindow` + 1 次连续 DMA 传输
- 比 `drawRGBBitmap` 更快（TFT_eSPI 针对 ESP32 优化过）
- 绘制 10 个汉字约 **20ms**

## 五、与独立 SPI 方案的对比

### 5.1 共享总线（本方案）

```cpp
// 屏幕和字库共用 FSPI（GPIO11/12/13）
// 优点：省引脚，接线简单
// 缺点：不能同时读写

TFT_eSPI tft;  // 默认 FSPI
// 字库也走 SPI（同一个总线）
SPI.beginTransaction(...);  // 切换参数
```

### 5.2 为什么不能定义 `USE_HSPI_PORT`（关键！）

如果 `User_Setup.h` 中定义了 `#define USE_HSPI_PORT`：
- TFT_eSPI 内部会使用 **HSPI（SPI3）** 外设
- 但代码中字库读取用的 `SPI` 全局对象是 **FSPI（SPI2）**
- 虽然引脚都是 GPIO11/12/13，但 GPIO Matrix 同一时间只能让一个外设控制引脚
- 结果：`SPI.transfer()` 操作被静默忽略，字库读取返回全零

**共享总线方案必须统一外设**：
- 去掉 `USE_HSPI_PORT` → TFT_eSPI 用 FSPI
- 字库读取也用 `SPI`（FSPI）→ 两者匹配

### 5.3 SPI 事务管理（不需要手动包裹）

`pushImage` 内部已自动调用 `begin_tft_write()` 和 `end_tft_write()`：

```cpp
// TFT_eSPI 内部实现：
void TFT_eSPI::pushImage(...) {
  begin_tft_write();     // spi.beginTransaction(MODE3) + CS_L
  setWindow(x, y, ...);
  pushPixels(data, len); // DMA 传输
  end_tft_write();       // CS_H + spi.endTransaction()
}
```

因此**不需要**外部包裹 `startWrite()` / `endWrite()`：

```cpp
// ❌ 多余的外部包裹
 tft.startWrite();
  tft.pushImage(x, y, 16, 16, pixBuf);
  tft.endWrite();

// ✅ 直接调用即可
  tft.pushImage(x, y, 16, 16, pixBuf);
```

### 5.4 独立 SPI（备选方案）

```cpp
// 屏幕用 FSPI，字库用 HSPI（完全独立）
// 优点：真正并行，互不干扰
// 缺点：代码复杂，需要两个 SPI 实例

SPIClass fontSPI(HSPI);  // 独立 SPI 实例
fontSPI.begin(TFT_SCLK, TFT_MISO, TFT_MOSI, FONT_CS);
```

**不推荐**：共享总线更简单可靠。独立 SPI 仅在需要并行高速传输时使用。

## 六、常见问题

### 6.1 编译报错 "User_Setup.h not found"

确保：
1. 已安装 TFT_eSPI 库（Arduino IDE 库管理器）
2. `User_Setup.h` 已正确配置并保存
3. 重启 Arduino IDE

### 6.2 屏幕颜色反相

TFT_eSPI 默认已处理 IPS 反色。如果颜色仍反：
```cpp
tft.init();
// tft.invertDisplay(true);  // 需要时手动开启
```

### 6.3 字库读取返回全零

排查顺序：
1. 确认 `User_Setup.h` 中 `TFT_MISO = 13`（字库数据输出）
2. 确认 GPIO9（CS2）接线正确
3. 用示波器看 CS2 拉低后 SCK 是否有 8MHz 时钟
4. 降低字库 SPI 频率到 1MHz 测试

### 6.4 汉字显示为乱码

- 检查 GB2312 地址计算（+846 偏移不能漏）
- 确认代码文件保存为 UTF-8 时，`\xHH` 转义仍能正确解析
